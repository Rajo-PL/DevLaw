# Role w Ekosystemie DevLaw

### 1. Lex-Architekt (Inżynier Prawa)
- **Zadania:** Projektowanie modułowe, pisanie Smart Contracts w Solidity/Python, zarządzanie repozytorium Git-Law [28, 29].
- **Wymogi:** Logika formalna, algorytmika, umiejętność przeprowadzania Stress-Testów [30, 31].

### 2. Sędzia-Audytor (Debugger)
- **Zadania:** Analiza logów systemowych (`Stack Trace`), obsługa wyjątków (`Exceptions`), raportowanie błędów do inżynierów (Wyrok jako Hotfix) [32, 33].
- **Uprawnienia:** Techniczny reset transakcji, nadpisanie decyzji automatu [34].

### 3. Przysięga Inżyniera Prawa
*"Będę traktować Konstytucję jako nienaruszalne Jądro Systemu... Nigdy nie wdrożę przepisu, którego nie poddałem rygorystycznym testom... Będę chronić dane obywateli wewnątrz logiki algorytmów."* [35-37].
